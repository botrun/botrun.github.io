HTML 投影片技能

> 將任意主題製作為矽谷流敘事簡報 HTML，純靜態單檔，可下載 PDF。

何時使用

- 用戶說「簡報」「投影片」「slides」「HTML 簡報」
- 用戶提供主題 + 要求視覺化呈現
- 用戶要求矽谷流說故事風格

設計原則

矽谷流敘事（必遵守）

遵循《矽谷流萬用敘事簡報法則》四步驟：

1. 設定問題（痛點） — 讓觀眾感受到問題的存在
2. 呈現數據（洞見） — 用一個核心訊息翻轉認知
3. 給出解方（架構） — 視覺化架構圖
4. 行動建議（CTA） — 明確下一步

視覺風格（預設雅緻淺色）

| 項目 | 規範 |
|------|------|
| 底色 | 淺暖白 #faf8f5，卡片白 #ffffff |
| 字體 | 標題 Noto Serif TC（襯線），內文 Noto Sans TC（無襯線） |
| 色彩 | 3 色為限：主色（珊瑚 #e8634a）、輔色（青 #0d9488）、點綴（紫 #7c3aed） |
| 字量 | 每頁最多 3 句話，數字放大、文字精簡 |
| 字級 | 標題 clamp(2.4rem, 5vw, 4rem)，lead clamp(1.1rem, 1.8vw, 1.4rem) |
| Icon | 禁止 emoji，一律用 SVG icon（Lucide 風格：24x24 viewBox, stroke-width 1.8, stroke-linecap/linejoin round） |
| 圖表 | 純 SVG 內嵌，禁止外部圖片/CDN |

技術規範

| 項目 | 規範 |
|------|------|
| 檔案 | 純靜態單一 HTML 檔案，零依賴 |
| 字體載入 | Google Fonts @import（Noto Serif TC + Noto Sans TC） |
| 頁數 | 預設 5 頁（可依需求調整，建議 3-7 頁） |
| 導航 | 鍵盤左右鍵 + 點擊箭頭 + 手機觸控滑動 |
| 動畫 | fade-up 進場動畫，CSS transition，無 JS 動畫庫 |
| PDF | 左上角「PDF」按鈕，使用 window.print() + @media print |
| RWD | 手機自動單欄，@media(max-width:768px) |

執行流程

Step 1：理解主題

1. 如果用戶提供了參考資料路徑，先讀取內容
2. 確認核心訊息、目標受眾、頁數

Step 2：規劃故事線

依矽谷流四步驟規劃每頁內容：

`
第 1 頁：痛點 — 一個讓觀眾有感的問題（配大數字）
第 2 頁：洞見 — 翻轉認知的核心觀點
第 3 頁：解方 — 架構圖 / 流程圖（SVG）
第 4 頁：實證 — 數據佐證（3 個關鍵指標）
第 5 頁：行動 — CTA + 時間軸 / 步驟圖
`

1. 使用 [模板](templates/base-template.html) 作為骨架
2. 輸出到 ./html/{主題名稱}.html

Step 5：部署 Vercel（必要步驟）

SVG 圖表指引

- 架構圖： 方塊 + 箭頭 + 標籤
- 流程圖：水平時間軸 節點 + 連線
- 對比圖：左右分區，顏色區隔（紅=舊/問題，綠=新/解方）
- 數據卡：大數字 font-size: 3.5rem; font-weight: 900 + 小標籤
- 人物圖：簡筆火柴人 頭 + 身體四肢
- 顏色使用 CSS 變數 var(--coral) var(--teal) var(--violet)
- marker 箭頭用 定義

@media print CSS（PDF 必備）

`css
@media print {
@page { size: landscape; margin: 0 }
body { background: #fff; overflow: visible }
.deck { display: block; width: auto; transform: none !important }
.slide { width: 100vw; height: 100vh; page-break-after: always; page-break-inside: avoid }
.slide:last-child { page-break-after: auto }
.fade-up { opacity: 1 !important; transform: none !important }
.nav, .arrow, .page-num, .pdf-btn { display: none !important }
}
`

永久地雷記錄

地雷 0：禁止 Emoji — 2026-03-16

- 問題：emoji 在 PDF 列印、跨平台顯示不一致，格調不夠
- 正確：所有 icon 一律用 inline SVG 繪製（Lucide 風格：24x24 viewBox, stroke-width 1.8, stroke-linecap/linejoin round, fill none）
- 驗證：全文搜尋確認零 emoji 字元

地雷 1：CSS 變數在 SVG 中的使用 — 2026-03-16

- 問題：SVG 內的 fill 和 stroke 使用 CSS 變數 var(--xxx) 時，部分瀏覽器 PDF 列印可能不渲染
- 解法：SVG 中盡量使用 CSS 變數，但確保 @media print 時背景為白色
- 驗證：列印預覽確認 SVG 文字和線條可見

地雷 2：Google Fonts @import 必須在 CSS 最頂部 — 2026-03-16
