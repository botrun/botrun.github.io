web-deploy

網頁發佈與建構。當用戶說「發佈網頁」「部署網站」「建構 web」「更新網頁」時使用此技能。自動掃描 projects/ 下的 HTML 並發佈到 GitHub Pages。

網頁發佈技能

何時使用

- 用戶說「發佈網頁」「部署網站」「更新網頁」
- 用戶新增了 HTML/CSS/JS 到 projects/ 目錄
- 用戶要求建構網頁索引

架構設計

`
timeless-bowen (私有 repo)
├── projects/ # 專案源碼（私有）
│ ├── 專案A/
│ │ └── *.html # 要發佈的網頁
│ └── 專案B/
└── web/ # 發佈目錄（GitHub Pages）
├── index.html # 自動生成的索引
├── assets/css/style.css # 共用樣式
└── projects/
├── 001-專案A/
└── 002-專案B/
`

URL 格式：https://{org}.github.io/{repo}/projects/{id}/

使用流程

步驟 1：建構網頁

執行建構腳本：

`bash
npx tsx .claude/skills/永恆知己-網頁發佈/scripts/build-web.ts
`

腳本會自動：
1. 掃描 projects/ 下所有 HTML 檔案
2. 複製到 web/projects/{專案ID}/
3. 複製相關 CSS/JS/圖片資源
4. 生成專案索引頁
5. 生成主索引頁 web/index.html

步驟 2：預覽（本地）

`bash
cd web && python3 -m http.server 8000
`

開啟 http://localhost:8000 預覽

步驟 3：提交並推送

`bash
git add web/
git commit -m "build: 更新網頁發佈內容"
git push
`

GitHub Actions 會自動部署到 GitHub Pages。

自動觸發條件

GitHub Actions 會在以下情況自動觸發部署：

- projects/*/.html 變動
- projects/*/.css 變動
- projects/*/.js 變動
- projects/*/.png|jpg|gif|svg|webp 變動
- web/** 變動
- 手動觸發（workflow_dispatch）

首次設定

需要在 GitHub repo 設定中啟用 GitHub Pages：

1. 進入 repo → Settings → Pages
2. Source 選擇 "GitHub Actions"
3. 完成！

檔案參考

- 建構腳本：[scripts/build-web.ts](scripts/build-web.ts)
- GitHub Actions：.github/workflows/deploy-pages.yml
- 共用樣式：web/assets/css/style.css
