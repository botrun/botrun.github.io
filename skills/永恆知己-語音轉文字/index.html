timeless-speech-to-text

永恆知己語音轉文字技能。當用戶放入音訊/影片檔案時自動觸發。使用 CLI pipeline + Gemini API (gemini-flash-latest) 進行高品質轉錄。轉錄完成後自動串接「會議記錄」技能。

永恆知己 - 語音轉文字技能

🎯 觸發條件

自動觸發（最重要！）：
- 用戶放入 .mp3, .m4a, .wav, .webm 等音訊檔案
- 用戶放入 .mp4, .mov, .avi, .mkv 等影片檔案
- 用戶說「處理會議錄音」

---

🎯 Pipeline 功能（只產生 .md 逐字稿）

單檔處理


`bash

音訊檔案


npx tsx scripts/pipeline.ts "收集箱/會議錄音/檔案名.m4a"

影片檔案（自動提取音訊）


npx tsx scripts/pipeline.ts "收集箱/會議錄音/會議.mp4"
`

批次處理（平行）


`bash
npx tsx scripts/pipeline.ts --batch 收集箱/會議錄音/ --jobs 4
`

Pipeline 自動執行


1. 影片先提取音訊（ffmpeg）🎬
2. Gemini 轉錄（gemini-flash-latest）
3. Gemini 語義分析（檔名+摘要+關鍵字）
4. 輸出 .md 逐字稿

---

⚠️ 重要：程式碼只做轉錄！

Pipeline 只產生 .md 逐字稿，不產生 .txt 會議記錄！

以下由 Claude Code Opus「會議記錄」技能處理：
- ❌ 禁止程式碼產生 .txt
- ❌ 禁止程式碼套用勘誤表
- ❌ 禁止程式碼格式化會議記錄

---

🔄 完整流程（兩技能串接）

1. 語音轉文字技能（本技能）
- 執行 pipeline
- 產出 .md 逐字稿
- 不打開任何檔案
- ⚡ 完成後自動觸發「會議記錄」技能

2. 會議記錄技能（總指揮）
- 讀取勘誤表：配置/規則/勘誤表.md
- 讀取 .md 逐字稿
- 手動套用勘誤校正
- 產出精華版 + 詳細版兩個 .txt
- 最後 open 同時打開兩個 .txt 讓用戶檢閱

📁 輸出與歸檔位置

| 類型 | 路徑 | 產生者 |
|------|------|--------|
| 逐字稿 .md | 素材庫/來源歸檔/會議記錄/逐字稿/ | Pipeline |
| 會議記錄（精華版）.txt | 素材庫/輸出/會議記錄/{檔名}-會議記錄.txt | 會議記錄技能 |
| 會議記錄（詳細版）.txt | 素材庫/輸出/會議記錄/{檔名}-詳細會議記錄.txt | 會議記錄技能 |
| 原始音檔 | 素材庫/來源歸檔/會議記錄/原始音檔/ | Pipeline |
| 原始影片 | 素材庫/來源歸檔/會議記錄/原始影片/ | Pipeline |

---

🔧 技術規格

- 轉錄模型：gemini-flash-latest（絕對不能錯）
- 影片音訊提取：ffmpeg（AAC 128kbps, 44.1kHz）
- 語義分析：Gemini 一次調用完成檔名+摘要+關鍵字
- 勘誤/格式化：⚠️ 由 Claude Code Opus 會議記錄技能處理，禁止程式碼處理

---

⚡ 效能

| 指標 | 數值 |
|------|------|
| 30 秒音檔 | ~20 秒完成 |
| 影片音訊提取 | ~5-10 秒 |

---

🗂️ 收集箱使用方式

1. 無腦丟檔案：將音訊或影片丟到 收集箱/會議錄音/
2. 執行 pipeline：npx tsx scripts/pipeline.ts --batch 收集箱/會議錄音/
3. 自動歸檔：原始檔案會自動移動到對應的歸檔目錄
4. 自動觸發會議記錄技能：產生精華版 + 詳細版兩個 .txt
5. 自動打開兩個 .txt：完成後用戶可檢閱
