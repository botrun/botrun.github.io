閃電知己技能。使用 rg 超高速檢索本地知識庫，透過國網 Ministral 14B 模型在 7 秒內回答問題。當用戶需要快速查詢本地記憶、需要極速回應時觸發。

閃電知己 - Lightning Expert

🎯 核心目標

TTFT（Time To First Token）< 7 秒：從用戶提問到開始回答，全程 7 秒內完成。

🚀 三步驟極速流程

`
用戶提問 → rg 檢索（1 秒）→ NCHC API（5 秒）→ 開始回答（< 7 秒）
`

💡 設計原則

1. rg 先過濾：使用 ripgrep 超高速檢索，1 秒內過濾出相關內容
2. 小模型快速推理：使用 Ministral 14B（國網 1.6 折），TTFT < 5 秒
3. 單次呼叫：避免多次 API 往返，降低延遲
4. 本地優先：優先搜尋 記憶核心/、索引/、素材庫/ 等本地知識庫

🚨 地雷與注意事項

地雷 1：rg 結果過多

問題：rg 可能回傳數千行結果，超過 API context window

解決：
- 使用 rg -l 先列出檔案清單，再精準讀取
- 使用 head -n 200 限制行數
- 優先搜尋索引目錄

地雷 2：API Key 寫死

問題：API Key 不應該寫死在程式碼中

解決：
- 從 .env 讀取
- 提供 .env.example 模板

地雷 3：TTFT 超時

問題：網路延遲、API 繁忙導致 TTFT > 7 秒

解決：
- 設定 timeout（10 秒）
- 降級策略：超時則回傳 rg 原始結果
- 快取常見問題的答案

地雷 4：關鍵字提取失敗

問題：無法從用戶問題中提取有效關鍵字

解決：
- 提供預設關鍵字清單
- 使用 Claude Code 內建能力協助提取
- 全域搜尋作為備案

地雷 5：多關鍵字檢索失敗 ⭐ (2026-02-12 修正)

問題：當傳入多個關鍵字（如「藍圖 產品 波盈盈」），rg 會把整個字串當作一個關鍵字搜尋，導致找不到結果

症狀：
`bash

錯誤：rg 搜尋 "藍圖 產品 波盈盈" 這個完整字串（找不到）


rg -C 5 -i "藍圖 產品 波盈盈" 索引/

正確：應該搜尋包含「藍圖」OR「產品」OR「波盈盈」的檔案


rg -C 5 -i "(藍圖|產品|波盈盈)" 索引/
`

解決：
- 修正 searchWithRg() 函式，將空格分隔的關鍵字轉換為正則表達式 OR 搜尋
- 例如："藍圖 產品 波盈盈" → "(藍圖|產品|波盈盈)"
- 單一關鍵字時不加括號，保持原樣

修正版本：
`typescript
// 多關鍵字處理：將空格分隔的關鍵字轉換為正則表達式 OR 搜尋
const keywordArray = keywords.trim().split(/\s+/).filter(k => k.length > 0);
const regexPattern = keywordArray.length > 1
? (${keywordArray.join('|')})
: keywordArray[0] || keywords;

const cmd = cd "${CONFIG.REPO_ROOT}" && rg -C 5 -i "${regexPattern}" "${targetDir}" 2>/dev/null || true;
`
