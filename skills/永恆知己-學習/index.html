timeless-learning

永恆知己學習技能。當用戶說「要」「不要」「好」「不好」「對」「錯」「喜歡」「不喜歡」時觸發。即時學習用戶偏好並寫入偏好紀錄。

永恆知己 - 學習技能

觸發條件

當用戶說以下關鍵字時，立即學習：

| 正向（要） | 負向（不要） |
|-----------|-------------|
| 我要、我想要 | 我不要、不想要 |
| 這個好、很好 | 這個不好、不行 |
| 對、沒錯 | 錯、不對 |
| 喜歡、偏好 | 不喜歡、討厭 |
| 繼續、就這樣 | 停、別這樣 |
| 保留、留著 | 刪掉、移除 |

學習流程

1. 偵測：用戶說了偏好關鍵字
2. 提取：提取具體偏好內容
3. 分類：正向（要）或負向（不要）
4. 寫入：記憶核心/語義記憶/偏好學習/偏好紀錄.md
5. 確認：回報「已記住您的偏好」

寫入格式

追加到 記憶核心/語義記憶/偏好學習/偏好紀錄.md：

`markdown

[YYYY-MM-DDTHH:mm:ss] [主題]

- 類型：正向/負向
- 內容：[具體偏好]
- 原話：「[用戶原話]」
- 情境：[當時在做什麼]
- 權重：10
`

範例

用戶說：「這個太囉嗦了，我不要這麼長的回答」

學習動作：
`markdown

2026-01-19T14:35:22 回答長度

- 類型：負向（不要）
- 內容：不要太長的回答
- 原話：「這個太囉嗦了，我不要這麼長的回答」
- 情境：AI 回答過長時
- 權重：10
`

回報：「已記住：您偏好簡短的回答，我會調整。」

偏好衝突處理

當新偏好與舊偏好衝突時：

1. 以最新為準：更新 updated 日期
2. 保留歷史：舊偏好加上 [已更新] 標記
3. 告知用戶：「您之前說 X，現在改為 Y，已更新。」

自動應用

學習後的偏好會自動影響：

1. 回答風格：長度、詳細程度、語氣
2. 搜尋排序：偏好相關的結果優先
3. 建議方式：避免用戶不喜歡的方式
