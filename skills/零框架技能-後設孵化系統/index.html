零框架技能 - 後設孵化系統

> 核心理念：泛化通用、禁止過度擬合、孵化技能後解決問題
>
> 適用範圍：所有機關專案（無預設立場）
>
> 相依自足：本目錄可獨立複製使用，所有引證檔案皆內建

---

何時使用此技能

✅ 主要用途（日常使用）

1. 即時問題解決：透過 Claude Code Web + HTTPS Proxy 解決用戶當場提問的任意問題
2. 技能孵化：將解決方案孵化進步，變成可複用的技能集和腳本集
3. 新專案啟動：任何領域的專案孵化（採購法、勞基法、環境法規...）
4. 完整流程：需要執行零幻覺四部曲（耳眼手心）

🎯 進階用途（大模型蒸餾小模型時才需要）

僅在以下情境才需要執行多模型評測（8 個模型 via OpenRouter API）：
- 大模型蒸餾小模型：將大模型的知識遷移到小模型
- 大模型智慧遷移小模型：優化小模型在特定領域的表現
- 產出排行榜報告：需要比較多個模型的表現差異

❌ 不適用情境

- 單一檔案處理（直接用 sh/cli.sh process）
- 簡單問答（不需要孵化技能）
- 已有現成技能可用

核心原則（三大禁令）

`yaml
禁止過度擬合:
- ❌ if (domain === '採購法') { specialLogic() }
- ❌ 硬編碼特定機關的處理邏輯
- ✅ 透過配置檔處理差異

禁止預設立場:
- ❌ 假設用戶是某種類型的專案
- ❌ 限定特定的資料格式
- ✅ 詢問用戶的具體需求
- ✅ 動態適應任何領域

禁止竄改來源:
- ❌ 修改原始文字內容
- ❌ 省略引證資訊
- ✅ 完整保留 URL、檔案路徑、頁碼、時間點
- ✅ 引用時標明來源
`

孵化流程（核心 SOP）

步驟 1：上網研究（WebSearch）

`yaml
目的: 收集公開資料、智庫報告、官方文件
執行:
- WebSearch 查詢「{領域} 2026 最新」
- 識別權威來源（政府、智庫、學術）
- 記錄所有來源 URL（不竄改）
輸出:
- 來源清單（URL + 權重 1-10）
- 重點摘要
`

步驟 2：格式轉換（耳）

`yaml
目的: 將任意格式轉為 AI 友善文字
支援格式:
文件: PDF, DOCX, XLSX, PPTX
圖片: PNG, JPG, GIF, WEBP
音訊: M4A, MP3, WAV
影片: MP4, MOV, AVI
網頁: HTTP/HTTPS URL
資料: JSON, YAML, CSV, XML
執行:
- sh/cli.sh ear process
輸出:
- Markdown 純文字
- 保留來源引證（檔案路徑、頁碼、時間點）
`

步驟 3：來源引證（絕對不竄改）

`yaml
引證格式:
URL: "[標題](https://...)"
檔案: "來源：{檔案名稱} 第 {頁碼} 頁"
音訊: "來源：{檔案名稱} {分:秒}"
影片: "來源：{檔案名稱} {時:分:秒}"

原則:
- 原文引用，不修改內容
- 明確標示來源位置
- 保留完整 URL
`

步驟 4：高品味題庫（眼）

`yaml
題庫三大特色:
鑑別度: 能區分不同模型表現（標準差 > 1.5）
口碑: 答案讓人覺得「哇真讚」（滿意度 > 8/10）
複雜情境: 真實用戶會問的問題（非簡單問答）

題目結構:
question: 複雜情境問題
answer_must_contain: 必含內容陣列
should_mention: 應提內容陣列
reference_source: 引證來源
difficulty: easy | medium | hard | expert
tags: 標籤陣列

執行:
- sh/cli.sh eye generate <領域> -n 20
`

步驟 5：多模型評測（眼）⚡ 選擇性

> 何時需要：僅在「大模型蒸餾小模型」或「大模型智慧遷移小模型」時執行
> 日常使用：跳過此步驟，直接用 Claude Code Web 解決問題即可

`yaml
評測模型清單（進階用途）:
- openai/gpt-oss-120b
- meta-llama/llama-4-maverick
- google/gemini-3-flash-preview
- mistralai/devstral-2512
- mistralai/ministral-14b-2512
- mistralai/ministral-8b-2512
- anthropic/claude-haiku-4.5
- anthropic/claude-sonnet-4

評審模型: google/gemini-3-flash-preview

兩批評測:
Batch 1: 無網路（測試內建知識）
Batch 2: 有網路（測試搜尋整合）

執行:
- sh/cli.sh eye run <領域>
`

步驟 6：九維度評分

`yaml
口訣: 真新全快省準人安穩

維度:
真（真實性）: 來源可追溯、可驗證
新（新穎性）: 使用最新資料
全（完整性）: 涵蓋必含內容
快（速度）: 回應簡潔有效
省（節省）: token 使用效率
準（精準度）: 答案正確
人（人本）: 易於理解
安（安全性）: 無誤導資訊
穩（穩定性）: 可重現

認證等級:
🥇 金標: ≥ 85 分 (94.4%)
🥈 銀標: ≥ 80 分 (88.9%)
🥉 銅標: ≥ 70 分 (77.8%)
`

步驟 7：技能孵化（手）

`yaml
目的: 建立領域專用 SKILL.md 和 .ts 腳本
輸出結構:
.claude/skills/{領域}-專用技能/
├── SKILL.md # 技能定義
├── 題庫.json # 高品味題庫
├── 評測報告.md # 評測結果
└── scripts/
├── cli.ts # 領域專用 CLI
├── process.ts # 格式處理器
└── evaluate.ts # 評測腳本

執行:
- sh/cli.sh hand run <領域>
`

步驟 8：世界模型預測（心）

`yaml
後設預測六大步驟:
1. 原始數據: 該收集什麼資料？
2. 方法學: 該用什麼方法建模？
3. 驗證: 該如何驗證準確性？
4. 模型搭建: 神經 + 符號混搭
5. 避免過擬合: 泛化驗證
6. 演化計算: 朝向目標最佳化

執行:
- sh/cli.sh heart predict <主題>
`

軟體工程原則

`yaml
BDD (行為驅動開發):
- 先定義驗收條件
- 再實作功能
- 驗證行為符合預期

TDD (測試驅動開發):
- 先寫測試
- 再寫程式碼
- 確保測試通過

SOLID:
- S: 單一職責（每個模組只做一件事）
- O: 開放封閉（擴展開放、修改封閉）
- L: 里氏替換（子類可替換父類）
- I: 介面隔離（介面要小而專注）
- D: 依賴反轉（依賴抽象不依賴具體）

DRY:
- 不重複自己
- 共用邏輯抽成函式
- 配置抽成設定檔
`

HTTPS Proxy 處理

`typescript
// OpenRouter API 配置
const openrouter = new OpenAI({
baseURL: "https://openrouter.ai/api/v1",
apiKey: process.env.OPENROUTER_API_KEY,
// Proxy 環境配置
httpAgent: process.env.HTTPS_PROXY
? new HttpsProxyAgent(process.env.HTTPS_PROXY)
: undefined,
});

// Web Search 啟用方式
const response = await openrouter.chat.completions.create({
model: "anthropic/claude-sonnet-4",
messages: [...],
max_tokens: 32768,
plugins: [{ id: "web", max_results: 5 }], // 啟用網路搜尋
});
