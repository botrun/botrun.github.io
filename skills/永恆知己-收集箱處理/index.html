timeless-inbox-processor

永恆知己收集箱處理技能。自動處理收集箱中的 PDF、Office、圖片等檔案，轉換成純文字永久記憶，並將原始檔案歸檔到適當位置。

永恆知己 - 收集箱處理技能

🎯 核心目的

自動化處理 收集箱/ 目錄中的非文字檔案：
1. 轉純文字 → 永久記憶到 記憶核心/語義記憶/外部文件/
2. 原始檔案歸檔 → 搬移到 素材庫/原始檔案/ 對應子目錄
3. 建立關聯 → 純文字版本包含原始檔案的本地路徑
4. 版本控制友善 → 純文字上 GitHub，原始檔案本地保留

---

📋 處理範圍

支援檔案類型

| 類型 | 副檔名 | 歸檔位置 |
|------|--------|----------|
| PDF | .pdf | 素材庫/原始檔案/PDF/ |
| Word | .doc, .docx | 素材庫/原始檔案/Office/ |
| Excel | .xls, .xlsx | 素材庫/原始檔案/Office/ |
| PowerPoint | .ppt, .pptx | 素材庫/原始檔案/Office/ |
| 圖片 | .jpg, .jpeg, .png, .gif, .bmp, .webp | 素材庫/原始檔案/圖片/ |
| 其他 | .epub, .rtf | 素材庫/原始檔案/其他/ |

音訊/影片特殊處理

音訊和影片檔案請使用「永恆知己-語音轉文字」+「永恆知己-會議記錄」技能串接處理。

🔄 處理流程

Step 1: 掃描收集箱

`bash

掃描收集箱中需要處理的檔案


find 收集箱/ -type f \( \
-name "*.pdf" -o \
-name ".doc" -o -name ".docx" -o \
-name ".xls" -o -name ".xlsx" -o \
-name ".ppt" -o -name ".pptx" -o \
-name ".jpg" -o -name ".jpeg" -o -name "*.png" -o \
-name ".gif" -o -name ".bmp" -o -name "*.webp" -o \
-name ".epub" -o -name ".rtf" \
\) ! -name ".DS_Store"
`

Step 2: 呼叫文字轉換技能

對每個檔案呼叫「永恆知己-文字轉換」技能進行轉換：
- 使用 Claude API 進行高品質提取
- 保留結構、表格、格式
- 輸出 LLM 友善的 Markdown

Step 3: 語義化命名並寫入記憶

轉換後的純文字檔案寫入 記憶核心/語義記憶/外部文件/：

`
記憶核心/語義記憶/外部文件/
├── Claude團隊-產品介紹-2026-02-10.md
├── 年度報告-財務分析-2026-02-10.md
└── ...
`

Step 4: 搬移原始檔案

`bash

依檔案類型搬移到對應目錄


mv 收集箱/example.pdf 素材庫/原始檔案/PDF/example.pdf
mv 收集箱/report.docx 素材庫/原始檔案/Office/report.docx
`

⚙️ .gitignore 設定

確保以下規則已加入 .gitignore：

`gitignore

PDF 檔案（全域排除）


*.pdf

Office 檔案（全域排除）


*.doc
*.docx
*.xls
*.xlsx
*.ppt
*.pptx

原始檔案歸檔目錄（本地保留）


素材庫/原始檔案/
`

結果：
- ✅ 純文字版本（.md）→ 上傳 GitHub
- ❌ 原始檔案（PDF/Office）→ 本地保留，不上傳

---

⚠️ 注意事項

1. 音訊/影片：不在此技能處理範圍，請使用語音轉文字技能
2. 機密檔案：若檔案含機密資訊，請先移到 機密/ 目錄
3. 大型檔案：超過 50MB 的檔案可能需要較長處理時間
4. 加密檔案：加密的 PDF/Office 無法自動處理
