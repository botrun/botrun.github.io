timeless-archive

永恆知己歸檔技能。將任意路徑的 PDF、Office、圖片等檔案歸檔到適當位置，並轉為 LLM 友善純文字。當用戶說「歸檔」「archive」「歸檔 done」或提供外部檔案路徑要求處理時觸發。

永恆知己 - 歸檔技能

核心目的

處理來自任意路徑（~/Downloads/、外部硬碟、桌面等）的檔案：
1. 讀取內容 → 用 Claude 多模態能力提取純文字
2. 寫入記憶 → 語義化命名存入 記憶核心/ 對應分類
3. 原始檔案歸檔 → 複製到 素材庫/原始檔案/ 對應子目錄
4. 不刪除原始檔案 → 只做 cp，不做 mv（來源可能是 Downloads 等非本 repo 位置）

與收集箱處理的差異

| | 歸檔技能（本技能） | 收集箱處理 |
|---|---|---|
| 來源 | 任意路徑（Downloads、外部） | 僅 收集箱/ 目錄 |
| 觸發 | 用戶指定檔案路徑 | 掃描收集箱 |
| 原始檔案 | cp（不刪來源） | mv（搬移） |

處理流程

Step 1: 識別檔案類型與來源

`
用戶提供: /Users/40gpu/Downloads/某報告.pdf
↓
判斷檔案類型 → PDF
判斷內容分類 → 產品策略 / 技術資源 / 外部文件 / ...
`

Step 2: 讀取並提取內容

- PDF → 用 Read 工具直接讀取（Claude 多模態）
- 圖片 → 用 Read 工具讀取（Claude Vision）
- Office → 嘗試 Read，fallback 用 pandoc/python 工具

重點：提取完整內容，保留結構、表格、格式，輸出 LLM 友善 Markdown。

Step 3: 分類判斷

根據檔案內容自動判斷歸入哪個記憶分類：

| 內容類型 | 記憶位置 |
|----------|----------|
| 產品需求/策略/會議決議 | 記憶核心/情節記憶/產品策略/ |
| 技術文件/API/架構 | 記憶核心/語義記憶/技術資源/ |
| 外部報告/論文/文件 | 記憶核心/語義記憶/外部文件/ |
| 個人行程/機票/收據 | 記憶核心/情節記憶/個人行程/ |
| 產品研究/競品分析 | 記憶核心/情節記憶/產品研究/ |
| 其他/不確定 | 記憶核心/語義記憶/外部文件/ |

Step 4: 寫入純文字記憶

檔名格式：{主題關鍵字}-{YYYY-MM-DD}.md

`markdown

{從內容推斷的語義化標題}

- 日期：YYYY-MM-DD
- 來源：{來源描述，如 Asana 匯出、外部報告}
- authority：{10=用戶提供, 5=外部文件, 3=AI 生成}
- 原始檔案：素材庫/原始檔案/{子目錄}/{檔名}

---

{提取的完整純文字內容，Markdown 格式}

---

關鍵字：{以逗號分隔的關鍵字，方便 rg 檢索}
`

Step 5: 複製原始檔案到歸檔位置

`bash

依檔案類型複製到對應子目錄


cp "{來源路徑}" "素材庫/原始檔案/PDF/{語義化檔名}.pdf"
cp "{來源路徑}" "素材庫/原始檔案/Office/{語義化檔名}.docx"
cp "{來源路徑}" "素材庫/原始檔案/圖片/{語義化檔名}.png"
`

歸檔子目錄對照：

| 副檔名 | 歸檔子目錄 |
|--------|----------|
| .pdf | 素材庫/原始檔案/PDF/ |
| .doc, .docx, .xls, .xlsx, .ppt, .pptx | 素材庫/原始檔案/Office/ |
| .jpg, .jpeg, .png, .gif, .bmp, .webp | 素材庫/原始檔案/圖片/ |
| 其他 | 素材庫/原始檔案/其他/ |

Step 6: 輸出歸檔報告

`
歸檔完成：

| 項目 | 位置 |
|------|------|
| 原始 PDF | 素材庫/原始檔案/PDF/{檔名}.pdf |
| 純文字記憶 | 記憶核心/{分類}/{語義化檔名}-{日期}.md |

內容摘要：{一句話摘要}
`

---

.gitignore 保障

以下副檔名已在 .gitignore 全域排除，原始檔案不會上傳 GitHub：

- *.pdf — 第 127 行
- .doc .docx .xls .xlsx .ppt .pptx — 第 130-136 行
- 素材庫/原始檔案/ — 第 141 行

結果：
- 純文字 .md → 上傳 GitHub，AI/LLM 隨時可存取
- 原始檔案 → 本地保留，不上傳

使用範例

範例 1: Asana 匯出 PDF


`
用戶：歸檔 /Users/40gpu/Downloads/● 2025-bowen - 波駿馬內建零框架系列 - Asana.pdf
Claude：讀取 PDF → 提取內容 → 判斷為產品策略
→ 寫入 記憶核心/情節記憶/產品策略/波駿馬內建零框架系列-Asana完成任務-2026-02-28.md
→ 複製 素材庫/原始檔案/PDF/波駿馬內建零框架系列-Asana-2026-02-28.pdf
`

範例 2: 外部技術報告


`
用戶：歸檔 ~/Downloads/2026-AI-Trends-Report.pdf
Claude：讀取 PDF → 提取內容 → 判斷為外部文件
→ 寫入 記憶核心/語義記憶/外部文件/AI趨勢報告-2026-02-28.md
→ 複製 素材庫/原始檔案/PDF/AI趨勢報告-2026-02-28.pdf
`

範例 3: 標註 done 的歸檔


`
用戶：歸檔 done: /path/to/file.pdf
Claude：同上流程，額外在純文字中標註「狀態：已完成」
`

注意事項

1. 只複製不刪除：來源檔案可能在 Downloads 等外部位置，絕不 rm 或 mv
2. 語義化檔名：不用原始亂碼檔名，從內容推斷有意義的中文名稱
3. 時間戳必須：所有 記憶核心/ 下的 .md 檔名必須含 -YYYY-MM-DD
4. 機密判斷：若內容含帳密/API Key，提醒用戶改放 機密/ 目錄
5. 大型 PDF：超過 20 頁的 PDF 分批讀取，確保完整提取
6. authority 判斷：用戶自己的文件 = 10，外部文件 = 5，AI 摘要 = 3
