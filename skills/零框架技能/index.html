零框架技能

> 核心理念：換位思考、三案擇優、技能孵化、智慧前處理、零幻覺引證、持續迭代

---

六大步驟

1. 換位

初步掃描資料，換位思考使用者潛在動機：

- 使用者真正想要什麼？
- 背後的需求是什麼？
- 有哪些隱含的期待？

2. 三案

提出以使用者為中心的驗收要件，設計三種替代方案：

| 方案 | 實作方法 | 優點 | 缺點 | 評分 (1-10) |
|------|----------|------|------|-------------|
| A | ... | ... | ... | ? |
| B | ... | ... | ... | ? |
| C | ... | ... | ... | ? |

選擇評分最高的方案執行。

3. 技能

上網查 Claude Code SKILL.md 官方寫法，建立技能：

- BDD（行為驅動開發）：先定義驗收條件
- TDD（測試驅動開發）：先寫測試再實作
- SOLID：單一職責、開放封閉、里氏替換、介面隔離、依賴反轉
- DRY：不重複自己

技能結構：
`
.claude/skills/{技能名稱}/
├── SKILL.md # 技能定義
└── scripts/
└── run.ts # TypeScript 處理腳本
`

4. 前處理

智慧檔案處理：

| 情境 | 處理方式 |
|------|----------|
| 大檔案 | 自動切片 + 斷點續做 |
| 圖片/PDF 辨識 | Azure Vision OCR API |
| 音訊轉文字 | Gemini API |
| 任意格式 | 轉為 AI 友善純文字 |

5. 引證

回應時必須引證來源，全力零幻覺證明：

| 來源類型 | 引證格式 |
|----------|----------|
| 網址 | [標題](https://...) |
| 檔案 | 來源：{檔案名稱} |
| 頁碼 | 第 {N} 頁 |
| 行數 | 第 {N} 行 |
| 時間點 | {分:秒} 或 {時:分:秒} |

原則：
- 來源文字不能竄改
- 完整保留原始 URL
- 標明確切位置（頁碼、行數、時間點）

6. 迭代

若使用者不滿意成果，可再次執行「三案打分」迭代進步：

- 重新提出三種替代方案
- 重新打分 1-10 選最佳
- 更新 SKILL.md 與 .ts 腳本
- 持續迭代直到使用者滿意
