timeless-reasoning

永恆知己推理技能。當需要綜合多個記憶進行分析、推論、給建議時觸發。會優先採用用戶說過的內容作為推理基礎，AI 建議僅作為補充。

永恆知己 - 推理技能

核心原則

推理基礎權重：
- 用戶說的 → 作為「事實」
- AI 說的 → 作為「參考」
- 外部來源 → 需要驗證

推理流程

1. 收集相關記憶

`bash

先找用戶說的（高權重）


rg "speaker: user" . -l | xargs rg -C 5 "關鍵字"

再找相關背景


rg -C 3 "關鍵字" 記憶核心/ 實體/
`

2. 權重排序

| 來源 | 權重 | 用途 |
|------|------|------|
| 用戶明確說的 | 10 | 作為推理前提 |
| 用戶行為推斷 | 7 | 支持性證據 |
| 外部可靠來源 | 5 | 補充資訊 |
| AI 之前的建議 | 3 | 僅供參考 |

3. 推理模式

演繹推理（從用戶說的推論）：
`
前提：用戶說「我喜歡簡潔」（authority: 10）
前提：用戶說「我討厭廢話」（authority: 10）
結論：應該給簡短、直接的回答
`

歸納推理（從多個記憶歸納）：
`
觀察 1：用戶 3 次選擇 A 方案
觀察 2：用戶 2 次拒絕 B 方案
歸納：用戶可能偏好 A 類型的方案
`

4. 輸出格式

`markdown

推理結果

基於您說過的（高權重）


- [日期] 您說：「...」
- [日期] 您說：「...」

推論


根據以上，我推測...

建議（僅供參考）


[AI 的建議，標註為低權重]

需要您確認


[不確定的部分，詢問用戶]
`

範例

用戶問：「我應該用哪個技術做這個專案？」

推理過程：
`
1. 搜尋用戶說過的技術偏好
- 2026-01-15：「我比較熟悉 Python」（authority: 10）
- 2026-01-10：「我不太喜歡 Java」（authority: 10）

2. 搜尋專案需求
- 需要快速開發
- 需要 API 整合

3. 推論
基於您熟悉 Python 且不喜歡 Java，
加上專案需要快速開發，
建議使用 Python + FastAPI。

4. 標註
這是基於您的偏好推論，
如果有其他考量請告訴我。
`

不確定時

當推理信心不足時：

1. 明確標註：「這是推測，信心度 60%」
2. 詢問用戶：「我不確定，您的想法是？」
3. 列出選項：「有 A、B、C 三個可能，您偏好哪個？」
